# テストリスト (Test Lists)

## このディレクトリについて

このディレクトリには、各機能の実装時に使用する**テストリスト**を保存します。

テストリストは、Kent BeckのTest-Driven Development（TDD）で提唱されている手法で、実装前に必要なテストケースを洗い出し、実装中の進捗を管理するためのドキュメントです。

## テストリストの特徴

- **実装計画**: 何をどの順番で実装するかの計画
- **進捗管理**: チェックボックスで実装状況を可視化
- **成長する**: 実装中に新しいテストケースを追加
- **揮発的**: 同じ機能でも実装アプローチによって変わる

## 命名規則

```
<機能番号>_<機能名>_v<バージョン>.md
```

**例:**
- `01_photo_upload_download_v1.md` - 初回実装
- `01_photo_upload_download_v2.md` - ドラッグ&ドロップ機能追加時
- `02_face_detection_v1.md` - 顔検出機能の初回実装

## 運用方針

### 同じSpecの実装を進める場合
既存のテストリストファイルを更新します（チェックボックスを進める）。

### Specに変更が入った場合
新しいバージョン番号でテストリストを作成します。

## 関連ドキュメント

各テストリストは `spec/features/` 配下の機能仕様に対応しています。

- **テストリスト**: **どう実装するか** (How) - 実装順序とタスク管理
- **機能仕様 (Spec)**: **何を実装するか** (What) - 機能要件と受け入れ基準
- **開発ガイド**: **開発プロセス** - [DEVELOPMENT.md](../DEVELOPMENT.md) を参照

### TDD開発フローとの関係

1. **Spec** (`spec/features/*.md`) で機能仕様と受け入れ基準を確認
2. **テストリスト** (`testlists/*.md`) で実装順序を決定
3. **3段階TDD** で実装（詳細は [DEVELOPMENT.md](../DEVELOPMENT.md#tdd開発フロー)）
