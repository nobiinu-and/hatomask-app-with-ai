# AI協働ガイド

このディレクトリは、AIエージェント（GitHub Copilot、Claude、ChatGPTなど）との効果的な協働を支援するための資料を管理します。

## 📁 ディレクトリ構成

```
ai/
├── README.md                    # このファイル
├── prompts/                     # プロンプトテンプレート集
│   ├── system/                  # AIの基本動作を定義
│   └── tasks/                   # タスク別の指示テンプレート
└── logs/                        # AI協働の作業ログ
```

## 🎯 使用目的

### 1. プロンプト管理（`prompts/`）

再利用可能なプロンプトテンプレートを体系的に管理します。

#### `prompts/system/` - システムプロンプト
AIの基本動作、役割、制約を定義する恒久的な指示:
- プロジェクトのアーキテクチャ知識
- コーディング規約
- 技術スタック詳細
- 禁止事項・制約

#### `prompts/tasks/` - タスクプロンプト
特定のタスク実行時に使用する指示テンプレート:
- 新機能実装の手順
- バグ修正のアプローチ
- リファクタリング指針
- コードレビュー観点

### 2. 作業ログ（`logs/`）

AIとの協働セッションを記録し、ナレッジを蓄積します:
- 実装した機能の経緯
- 遭遇した問題と解決策
- 設計判断の理由
- 将来の参考情報

## 📖 プロンプトの使い方

### システムプロンプトの適用

新しいAIセッション開始時に読み込ませる:

```
以下のシステムプロンプトに従ってください：

[prompts/system/配下の内容をコピー]
```

### タスクプロンプトの利用

具体的なタスク実行時にテンプレートを参照:

```
以下のタスクを実行してください：

機能名: 写真一覧表示
[prompts/tasks/implement-feature.md の内容に沿って依頼]
```

## 🔄 ログの記録方法

作業完了後、以下の形式で `logs/` にセッションログを保存します:

**ファイル名規則**: `YYYY-MM-DD_タスク概要.md`

**テンプレート**:
```markdown
# [タスク名]

**日時**: YYYY-MM-DD
**担当AI**: [GitHub Copilot / Claude / etc.]

## 目的
[何を実現したかったのか]

## 実施内容
[具体的に何をしたか]

## 技術的判断
[なぜその方法を選んだか]

## 遭遇した問題と解決策
[問題があれば記録]

## 参考情報
[関連するファイルやドキュメント]
```

## 🚀 クイックスタート

### 新しい機能を実装する場合

1. `prompts/system/` 配下のファイルをAIに読み込ませる
2. `prompts/tasks/implement-feature.md` のテンプレートを使用
3. 作業完了後、`logs/YYYY-MM-DD_機能名.md` にログを記録

### バグを修正する場合

1. `prompts/tasks/fix-bug.md` のテンプレートを参照
2. 既存の `logs/` から類似問題を検索
3. 修正完了後、ログに記録

## 💡 ベストプラクティス

### プロンプト作成のコツ

1. **明確な役割定義**: AIに何を期待するか明示する
2. **具体的な制約**: やってはいけないことを明記
3. **コンテキスト提供**: プロジェクト固有の情報を含める
4. **例示**: 期待する出力の例を示す

### ログ記録のコツ

1. **タイムリーに記録**: 作業直後に書く
2. **判断の理由を残す**: なぜその選択をしたか
3. **問題と解決策をセット**: 再発防止に役立つ
4. **検索しやすく**: 明確なタイトルとタグ

## 🔧 メンテナンス

- プロンプトは定期的に見直し、プロジェクトの進化に合わせて更新
- ログは月ごとにサブディレクトリに整理（`logs/YYYY-MM/`）
- 古い情報は `archive/` に移動
