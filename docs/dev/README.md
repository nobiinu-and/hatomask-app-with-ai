# 開発ドキュメント（Developer Docs）

このディレクトリは、開発者が迷子にならないための「入口」です。

- 開発タスク（Task01〜Task06）の正本は `docs/ai/prompts/tasks/`（01〜06）です。
- この `docs/dev/` には、タスクの詳細手順を重複して書きません（変更コスト削減のため）。

## どこを読めばいい？

- **開発の進め方（手順）**: `howto/`
  - `howto/development.md`
  - `howto/docker.md`
  - `howto/ci.md`
- **規約（必須ルール）**: `standards/`
  - `standards/coding.md`
  - `standards/quality.md`
  - `standards/openapi.md`
  - `standards/linting.md`
- **ガイドライン（考え方）**: `guidelines/`
  - `guidelines/vertical-slice.md`
- **ポリシー（取り扱い/禁止事項）**: `policies/`
  - `policies/data-handling.md`
