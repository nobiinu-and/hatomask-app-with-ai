# 写真アップロード・ダウンロード機能 ドメインモデル

## 概要
ユーザーが写真（JPEG/PNG）をアップロードし、プレビュー表示、ダウンロードできる最小限の機能のドメインモデル。  
**機能範囲**: 写真アップロード（ファイル選択）、プレビュー表示、ダウンロード（無加工）

## エンティティ

### Photo
**説明**: アップロードされた写真を表現するエンティティ。画像データ、メタデータ、タイムスタンプを管理する。

**プロパティ**:
- id: UUID (自動生成)
- originalFileName: String (必須、元のファイル名を保持)
- contentType: String (必須、"image/jpeg" または "image/png")
- fileSize: Long (必須、バイト単位のファイルサイズ、最大10MB)
- imageData: byte[] (必須、画像ファイルの実データ)
- createdAt: LocalDateTime (自動生成)
- updatedAt: LocalDateTime (自動更新)

**バリデーションルール**:
- contentType: "image/jpeg" または "image/png" のみ許可
- fileSize: 10MB (10,485,760 bytes) 以下
- originalFileName: 空文字列・null不可、最大255文字
- imageData: 空・null不可

**ビジネスルール**:
- 一度アップロードされた Photo は無加工でダウンロード可能
- 同一ユーザーが複数回アップロードした場合、新しい Photo エンティティが作成される（上書きではなく追加）

## バリューオブジェクト

### ContentType
**説明**: 画像ファイルのMIMEタイプを表現する値オブジェクト。許可された形式のみを受け入れる。

**プロパティ**:
- value: String ("image/jpeg" または "image/png")

**バリデーションルール**:
- value は "image/jpeg" または "image/png" のいずれか
- 空文字列・null不可

### FileSize
**説明**: ファイルサイズを表現する値オブジェクト。最大サイズ制限を保証する。

**プロパティ**:
- bytes: Long

**バリデーションルール**:
- bytes は 1 以上、10,485,760 (10MB) 以下
- 負の値・null不可

## リポジトリインターフェース

### PhotoRepository
**説明**: Photo エンティティの永続化を担当するリポジトリ

**メソッド**:
- save(photo: Photo): Photo
  - Photo を保存し、ID、createdAt、updatedAt が設定されたインスタンスを返す
  - 新規作成時は createdAt と updatedAt を現在時刻で設定
  - 更新時は updatedAt のみ現在時刻で更新
- findById(id: UUID): Optional<Photo>
  - ID で Photo を検索する
  - 存在しない場合は Optional.empty() を返す
- deleteById(id: UUID): void
  - ID で Photo を削除する
  - 存在しない場合は何もしない

## 補足・注意事項

### モデリング時の考慮点
1. **Specに記載された機能のみを実装**:
   - ユーザー管理、アルバム、タグ、画像編集などは含まない
   - 将来の拡張候補として記載されているが、現時点では実装しない

2. **シンプルな永続化**:
   - 画像データは imageData プロパティに byte[] として保存
   - 将来的にクラウドストレージ（S3など）に移行する可能性があるが、現時点では直接保存

3. **複数アップロード時の動作**:
   - ユーザーが複数回アップロードすると、それぞれ新しい Photo エンティティが作成される
   - フロントエンドは最新の Photo ID のみを保持してプレビュー・ダウンロードを行う
   - 古い Photo の削除はこのフェーズでは考慮しない

4. **バリデーション責務**:
   - ファイルサイズとMIMEタイプのバリデーションは値オブジェクト（ContentType, FileSize）で保証
   - Photo エンティティは有効な値オブジェクトを受け取る前提

### Specに記載された機能範囲の確認
- ✅ 写真アップロード（JPEG/PNG、最大10MB）
- ✅ プレビュー表示（Photo ID を使用してデータを取得）
- ✅ ダウンロード機能（Photo の imageData をそのまま返却）
- ❌ ユーザー認証・管理（Specに記載なし）
- ❌ アルバム・コレクション（Specに記載なし）
- ❌ タグ付け機能（Specに記載なし）
- ❌ 画像編集（ハトマスク適用は将来の拡張候補）
