## ロードマップ（4 段階＋拡張）

### Milestone 0：静止画での 3D マスク適用 — "技術検証とコア体験の確立"

**狙い**：リアルタイム追従の前に、静止画で基本機能を確立／技術的な実現可能性を検証

- 写真アップロード →**1 人の顔検出**→ 3D ハトマスク適用（静止画）
- マスクの向き：顔の角度に合わせた 3D 配置（首の回転・傾きを反映）
- 出力：**処理済み画像の保存（端末へダウンロード）**
- 調整：ズレた時の最小調整（位置/スケールくらい）

**データ取り扱い（M0 方針 / Option B）**

- サーバが画像を受信する可能性は許容する（題材として検討する）
- ただし、実在写真（入力由来ベース）および派生データは **非永続化・短命 TTL・非ログ** を必須とする
- 共有は Milestone 0 の必須要件ではない（Milestone 1 以降で扱う）

**やらない**：リアルタイム追従、動画、種類、ガチャ、美化、複雑な UI

**このマイルストーンの意義**：

- 顔検出ライブラリ（MediaPipe）の動作確認
- Three.js での 3D マスク描画の実装検証
- UI/UX の初期フィードバック取得
- 技術的な課題の早期発見

---

### Milestone 1：リアルタイム 3D 追従（1 人）— "気持ちよさを実現"

**狙い**：Time to first 追従が速い／動いて楽しい／成果物を持ち帰れる

- カメラ起動 →**1 人の顔をリアルタイム検出**→ 3D ハトマスクが追従
- 追従：首の回転・傾き（リアルタイム、60fps 目標）
- 出力：**短い動画 or 画像キャプチャ＋共有**（ここが「プロダクト」化の鍵）
- 調整：Milestone 0 で確立した調整機能を継承

**やらない**：種類、ガチャ、美化、スタンプ量産、複雑な UI

---

### Milestone 2：3D 追従（複数人）— “みんなでやると強い”に広げる

**狙い**：集合写真・複数人動画が作れる → 利用シーンが増える

- 画面内の**複数の顔を同時追従**
- 優先順位（例）：最大顔を優先／全員追従／上限 2〜4 人など（実装負荷で選ぶ）
- 出力：複数人状態での保存・共有
- 安定性：追従の破綻を減らす（入れ替わり/ロスト時の挙動を決める）

**やらない**：美化の凝り、種類の大量追加、ガチャ運用

---

### Milestone 3：見た目の価値を増やす — 種類 → 美化（順番このままで OK）

**狙い**：「飽きない」「映える」を足して継続率を上げる

**3A. 鳩の種類（スキン）**

- ドバト / キジバト / 白鳩 / 伝書鳩 / サイバー / スチームパンク…
- まずは**切替**（ガチャは後で）
- “似合う”おすすめ（任意、後回し可）

**3B. ワンタップ美化**

- 目キラ、羽ツヤ、くちばし質感、軽いライティング
- 強度スライダー or プリセット（ナチュラル/盛る/ガチ盛り）

---

## その先の拡張（今は保留で OK）

- **ハト度スライダー**：ちょいハト〜ガチハト調整（共有の恥ずかしさを下げる）
- **鳩スタンプ生成（16 種）**：量産・LINE 用途へ（別軸の強み）
- **ガチャ**：継続/収集の運用要素（最後に入れると安全）
